# Factorio Server Pro - 项目开发框架

## 项目概述
Factorio Server Pro 是一个用于管理 Factorio 游戏服务器的 Web 管理系统。

## 技术栈
- **后端**: PHP 7.4+
- **数据库**: SQLite
- **前端**: HTML5 + JavaScript + jQuery
- **实时通信**: Workerman WebSocket
- **游戏交互**: Factorio RCON

## 目录结构
```
factorio/
├── web/
│   ├── app/
│   │   ├── controllers/      # 控制器层
│   │   ├── core/             # 核心类
│   │   ├── helpers/          # 辅助函数
│   │   ├── modules/          # 功能模块
│   │   ├── public/           # 公共资源
│   │   │   ├── assets/       # 前端资源
│   │   │   └── pages/        # HTML 页面
│   │   ├── services/         # 服务层
│   │   └── api.php           # API 入口
│   ├── config/               # 配置文件
│   ├── data/                 # 数据目录
│   │   └── factorio.db       # SQLite 数据库
│   └── logs/                 # 日志目录
├── server/                   # Factorio 服务端
├── versions/                 # Factorio 版本
└── mods/                     # 模组目录
```

## 核心模块
### 1. 用户认证模块
- 文件: `app/controllers/AuthController.php`
- 服务: `app/services/AuthService.php`

### 2. 服务器管理模块
- 文件: `app/controllers/ServerController.php`
- 服务: `app/services/ServerService.php`, `app/services/RconService.php`

### 3. 商店系统
- 文件: `app/controllers/ShopController.php`
- 服务: `app/services/ShopService.php`

### 4. VIP 系统
- 文件: `app/controllers/VipController.php`
- 服务: `app/services/VipService.php`

### 5. 自动响应守护进程
- 文件: `app/modules/autoResponder/daemon.php`
- 功能: 玩家欢迎、聊天响应、投票系统、物品请求

### 6. 状态管理
- 文件: `app/services/StateService.php`
- 功能: 统一管理应用状态，支持数据库和文件两种模式

## 数据库
- 类型: SQLite
- 文件位置: `web/data/factorio.db` (使用相对路径引用)
- 主要表:
  - users: 用户表
  - shop_items: 商店商品
  - orders: 订单表
  - player_histories: 玩家历史
  - votes: 投票记录
  - chat_trigger_responses: 聊天触发响应

## API 规范
- 入口: `/app/api.php`
- 认证: Session-based
- 响应格式: JSON
- 公共 Actions: 无需登录的接口在 `api.php` 中 `$publicActions` 数组定义

## 前端规范
- 框架: 原生 JavaScript + jQuery
- 样式: Bootstrap 5
- 图标: Bootstrap Icons
- 页面: 所有 HTML 页面位于 `app/public/pages/`

## 开发规范
1. 数据库操作使用 `App\Core\Database` 类
2. 状态管理优先使用 `StateService`
3. API 响应使用 `App\Core\Response` 类
4. 所有测试文件放在 `test/` 文件夹，以 `test` 开头
5. 任务完成后删除临时测试文件
