## 二、V2 新增模块说明

### 2.99 AutoResponder 模块（V2 数据库化）

自动响应系统在 V2 版本中完成了彻底的数据库化改造，通过 StateService 适配器统一访问数据库。

#### 改动点

- **构造函数**支持 StateService 注入（可选参数，向后兼容）
- **loadSettings()** 通过 `StateService::loadState('chatSettings')` 从数据库读取
- **saveSettings()** 通过 `StateService::saveState()` 写入数据库
- **loadState()/saveState()** 使用 `auto_responder_states` 表

#### 向后兼容性

- 无参调用仍然有效（内部自动实例化 StateService）
- 数据库不可用时自动回退到 JSON 文件
- 公共接口完全不变（start/stop/sendMessage 等）

#### 数据库表映射

| 功能模块 | 数据库表 | 说明 |
|---------|----------|------|
| 周期性消息 | chat_scheduled_tasks | 定时任务配置和状态 |
| 关键词回复 | chat_trigger_responses | 触发器和响应内容 |
| 玩家事件 | chat_player_events | 欢迎/告别/礼包配置 |
| 运行状态 | auto_responder_states | 守护进程运行时状态 |
| 消息队列 | message_queues | 待发送消息队列 |

#### 异常类

V2 版本新增了异常层次结构，提供更精确的错误处理：

| 异常类 | 继承自 | 触发场景 | 特殊属性 |
|--------|--------|----------|----------|
| `DatabaseException` | \Exception | 数据库操作基础异常 | `$sql`, `$params`, `$code` |
| `ConnectionException` | DatabaseException | 连接失败或超时 | `$retryCount`, `$maxRetries` |
| `QueryException` | DatabaseException | SQL 执行错误 | `$sql`, `$params`, `$errorCode` |
| `ConstraintException` | QueryException | 外键/唯一约束违反 | `$constraintName`, `$table` |

**使用示例：**
```php
try {
    $db->query("SELECT * FROM nonexistent_table");
} catch (QueryException $e) {
    echo "SQL 错误: " . $e->getMessage();
    echo "错误代码: " . $e->getCode();
    echo "执行的 SQL: " . $e->getSql();
}

// 使用便捷事务包装器
$result = $db->transactional(function($db) {
    $db->execute("INSERT INTO orders ...");
    $db->execute("UPDATE inventory ...");
    return true;
});
```

### 2.1 StateService (services/StateService.php)

状态数据管理服务，统一管理所有运行时状态数据。V2 版本升级为双模式适配器，优先使用 SQLite 数据库存储，支持 JSON 文件回退。

#### 核心方法

| 方法 | 访问级别 | 参数 | 返回值 | 说明 |
|------|----------|------|--------|------|
| `__construct()` | public | $configDir: string | void | 构造函数 |
| `loadState()` | public | $name: string | array | 加载状态（优先 DB，回退文件） |
| `saveState()` | public | $name: string, $state: array | bool | 保存状态（双写 DB+文件） |
| `getStateValue()` | public | $stateName: string, $key: string, $default = null | mixed | 获取键值 |
| `setStateValue()` | public | $stateName: string, $key: string, $value: mixed | void | 设置键值 |
| `deleteState()` | public | $name: string | bool | 删除状态 |
| `clearCache()` | public | $name: string | void | 清除缓存 |

#### 表映射规则

V2 版本将状态名称映射到具体的数据库表：

| 状态名称 | 目标表 | 说明 |
|---------|--------|------|
| chatSettings | chat_* 表组 | 聊天设置（拆分为 4 张表） |
| autoResponderState | auto_responder_states | 自动响应运行状态 |
| voteState | votes + vote_records | 投票状态 |
| voteCooldown | vote_cooldowns | 投票冷却 |
| itemRequestConfirm | item_requests (status=confirm) | 物品请求确认 |
| requestItemCooldown | item_requests (status=cooldown) | 物品请求冷却 |
| messageQueue | message_queues | 消息队列 |
| logStreamState | log_stream_states | 日志流状态 |
| playerHistory | player_histories | 玩家历史记录 |

#### 降级策略

- **正常模式**：100% 使用 SQLite 数据库
- **DB 故障时**：自动切换到 JSON 文件（已归档到 _archive/ 目录）
- **降级事件**会记录警告日志（含状态名和文件路径）
- **兼容性**：旧版 JSON 文件仍然可读，确保向后兼容
