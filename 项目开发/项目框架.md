# FactorioWeb 项目框架文档

> **最后更新**: 2026-04-05
> **版本**: v6.0

---

## 一、项目概述

FactorioWeb 是一个 Factorio 游戏服务器 Web 管理面板，提供服务器控制、Mod 管理、玩家管理、配置编辑、实时监控等功能。

### 技术栈

| 层级 | 技术 |
|------|------|
| **后端** | PHP 7.4+ (自定义 MVC 架构) |
| **WebSocket** | Workerman |
| **前端** | HTML + JavaScript + Bootstrap 5 |
| **HTTP 客户端** | Axios |
| **终端模拟** | XTerm.js |
| **配置存储** | JSON 文件 |
| **进程管理** | Screen + RCON 协议 |

### 3.2 服务层

| 服务 | 职责 |
|------|------|
| Database | SQLite 数据库连接管理、初始化、查询封装 |
| ServerService | 服务器启动/停止/状态检测/RCON通信 |
| RconService | Factorio RCON 协议实现 |
| AuthService | 用户认证/密码加密/Session管理 |
| ChatService | 聊天设置/自动响应配置 |
| FileService | 文件上传/下载/管理 |
| MonitorService | 系统资源监控 |
| UserService | 用户注册、查询、密码修改 |
| ShopService | 商品管理、订单创建/查询/发放、价格计算 |
| ItemService | 物品数据加载、分类查询、搜索、物品请求管理 |
| ItemSyncService | 物品数据远程同步、同步状态管理 |
| VipService | VIP 等级管理、权限验证、折扣计算、自动续期 |

### 3.3 自动响应模块

功能包括：
- **周期性消息**: 定时发送服务器公告
- **关键词回复**: 自动回复玩家聊天
- **玩家事件**: 上线欢迎、下线告别
- **投票踢人**: 玩家投票踢出功能
- **物品请求**: 玩家可请求物品

## 十、数据库结构

### 10.1 数据库信息

- **类型**: SQLite 3
- **文件路径**: `web/data/factorio.db`
- **连接方式**: PDO (通过 Database 单例类)
- **日志模式**: WAL (Write-Ahead Logging) - 提升并发读写性能
- **外键约束**: 已启用 (PRAGMA foreign_keys = ON)
- **数据库版本**: V2 (2026-04-05 迁移完成)

### 11.3 故障排查指南

#### 常见问题

**问题 1: 数据库锁定错误**
```
SQLSTATE[HY000]: General error: 5 database is locked
```
原因：并发写入过多或长事务未提交
解决：
- 检查是否有未关闭的事务
- 增加 busy_timeout（当前 5 秒）
- 减少长时间运行的批量操作

**问题 2: WAL 文件过大**
现象：factorio.db-wal 文件超过 100MB
原因：checkpoint 未及时执行
解决：
```sql
-- 手动执行 checkpoint
PRAGMA wal_checkpoint(TRUNCATE);
```
或等待自动触发（每 100 个请求概率执行）

**问题 3: 数据库损坏**
现象：`PRAGMA integrity_check` 返回非 "ok"
原因：磁盘故障、断电、进程被杀
解决：
1. 从备份还原
2. 使用 `.recover` 命令尝试修复
3. 如无法修复，从最新备份恢复

### 11.4 维护命令速查

```bash
# 完整性检查
sqlite3 web/data/factorio.db "PRAGMA integrity_check;"

# 外键检查
sqlite3 web/data/factorio.db "PRAGMA foreign_key_check;"

# 表空间分析
sqlite3 web/data/factorio.db "ANALYZE;"

# 数据库优化（重建索引、清理碎片）

SELECT name,
       ROUND(pgsize * COUNT(*) / 1024.0, 2) || ' KB' as size
FROM dbstat GROUP BY name ORDER BY size DESC;
"
```

## 八、自动响应系统

自动响应系统随服务器启停联动运行，无需手动启动或停止。管理员只需在 Web 界面配置功能模块（周期消息、关键词回复、玩家事件、服务器响应等），系统会在服务器运行期间自动执行。

### 8.1 功能概述

| 功能 | 说明 |
|------|------|
| 周期性消息 | 按设定时间间隔自动发送消息（15分钟/30分钟/每小时/每2小时/每6小时/每12小时/每天） |
| 定时任务 | 在指定时间点发送一次性消息（活动预告、维护通知等） |
| 关键词自动回复 | 监听游戏内聊天，当玩家发送包含特定关键词时自动回复 |
| 玩家事件响应 | 玩家加入/离开时自动发送欢迎/告别消息 |

### 8.2 支持的变量

| 变量 | 说明 |
|------|------|
| `{player}` | 发送消息的玩家名 |

### 8.3 安装步骤

**方法一：使用自动设置脚本（推荐）**

```bash
cd /www/wwwroot/factorio/web
./setupCron.sh
```

选择选项 1 安装 Cron 任务。

**方法二：手动设置**

```bash
crontab -e
# 添加以下行（每分钟执行一次）
* * * * * /usr/bin/php /www/wwwroot/factorio/web/autoResponder.php >> /www/wwwroot/factorio/web/autoResponder.log 2>&1
```

### 8.4 配置示例

**周期性消息**
- 消息内容：`欢迎游玩本服务器！请遵守游戏规则。`
- 间隔：每小时

**关键词回复**
- 关键词：`帮助`
- 响应内容：`{player}，请查看服务器公告或联系管理员获取帮助。`

**玩家事件**
- 上线欢迎：`欢迎 {player} 加入游戏！祝你游戏愉快！`
- 下线告别：`{player} 离开了游戏，再见！`

### 8.6 故障排除

**自动响应不工作**
```bash
# 检查 cron 任务
crontab -l

# 查看日志
tail /www/wwwroot/factorio/web/logs/autoResponderDaemon.log

# 手动测试
php /www/wwwroot/factorio/web/autoResponder.php
```

**关键词回复不触发**
1. 确保关键词匹配（支持部分匹配）
2. 检查日志中是否有聊天消息被正确解析
3. 确认 `factorio-current.log` 文件路径正确

**玩家事件不触发**
1. 确认玩家事件功能已启用
2. 检查日志中是否有 `[JOIN]` 和 `[LEAVE]` 标记
3. 确认消息内容不为空
